7. Formato JWT¶

7.1. Componentes¶

Cada refresh/access token es un JWT es un objeto JSON codificado que posee tres partes:

  • header,
  • payload,
  • signature.

Lo siguiente es un access token generado en formato JWT utilizando la codificación descrita

Si inspeccionamos su contenido desde jwt.io, y exploramos su contenido:

7.1.1. Header¶

{
  "alg": "HS512"
}

La cabecera codifica el formato del token:

alg
el algoritmo utilizado para codificar el token. Debería ser “HS512”.
typ
el tipo del token. Ej. “JWT”.

7.1.2. Payload¶

{
  "sub": "0900315474",
  "exp": 1448404734,
  "iat": 1448403534,
  "iss": "zimple.com.py",
  "jti": "5BAZTFPJP5LZKWNZM5TA",
  "aud": "daaa",
  "mibi": {
    "auths": null,
    "type": "END_USER_ACCESS"
  }
}

El cuerpo describe la autorización concedida:

jti
identificador único del token.
iss
emisor del token.
sub
identificador de la cuenta que se provee acceso.
exp
la marca de tiempo de cuando el token expira.
mibi
la lista de permisos concedidos que posee el token.
auths
credenciales proporcionadas
type:
el tipo de acceso

7.1.3. Signature¶

La firma es un hash que sirve para verificar la autenticidad del token.
